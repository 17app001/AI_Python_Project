## 必懂功能

### 1. 並行工具執行

**功能說明**：同時執行多個操作以達到最大效率

**使用時機**：
- 一次讀取多個檔案
- 同時執行多個搜尋
- 並行資料處理

**範例**：
```python
# 不要循序執行：
read_file('file1.py')
read_file('file2.py')
read_file('file3.py')

# 使用並行執行：
read_file('file1.py')  # 全部同時執行
read_file('file2.py')
read_file('file3.py')
```

**最佳實踐**：
- 用於獨立操作
- 避免相依操作（A 的輸出是 B 的輸入）
- 預設使用並行，除非需要循序

### 2. 待辦清單管理

**功能說明**：自動任務規劃、追蹤與完成

**主要功能**：
- 從使用者請求自動生成任務
- 進度追蹤與狀態更新
- 基於優先級的任務組織
- 完成驗證

**使用模式**：
```python
# 自動任務創建
todo_list([
    {"id": "1", "content": "研究資料來源", "status": "pending", "priority": "high"},
    {"id": "2", "content": "實作核心邏輯", "status": "pending", "priority": "high"},
    {"id": "3", "content": "測試功能", "status": "pending", "priority": "medium"}
])

# 更新進度
todo_list([
    {"id": "1", "content": "研究資料來源", "status": "completed", "priority": "high"},
    {"id": "2", "content": "實作核心邏輯", "status": "in_progress", "priority": "high"},
    {"id": "3", "content": "測試功能", "status": "pending", "priority": "medium"}
])
```

### 3. 記憶系統

**功能說明**：專案特定資訊的持久化上下文存儲

**記憶類型**：
- **全域規則**：系統級指導原則
- **使用者記憶**：明確的使用者偏好
- **系統記憶**：從對話自動檢索的上下文

**使用方式**：
```python
# 存儲重要專案資訊
create_memory(
    Title="專案架構",
    Content="使用 MVC 模式，Flask 後端和 React 前端",
    Tags=["架構", "flask", "react"],
    CorpusNames=["my_project"]
)
```

### 4. 多重編輯操作

**功能說明**：在一個原子操作中對單一檔案進行多項變更

**使用時機**：
- 重構檔案的多項相關變更
- 同時新增匯入和實作函式
- 更新配置檔案的多項設定

**範例**：
```python
multi_edit(
    file_path="src/main.py",
    edits=[
        {"old_string": "import os", "new_string": "import os\nimport sys"},
        {"old_string": "def old_function():", "new_string": "def new_function():"},
        {"old_string": "return result", "new_string": "return processed_result"}
    ]
)
```

### 2. 錯誤處理與除錯

**系統性除錯流程**：
1. **識別根本原因**：不要只修復症狀
2. **新增日誌**：追蹤變數狀態和程式碼流程
3. **測試隔離**：建立最小測試案例
4. **迭代修復**：一次應用一項變更

**範例**：
```python
def debug_function(data):
    try:
        # 新增日誌
        logger.info(f"處理資料: {data}")
        
        # 驗證輸入
        if not data:
            logger.warning("收到空資料")
            return None
            
        # 處理並包含錯誤處理
        result = process_data(data)
        logger.info(f"處理成功: {result}")
        return result
        
    except Exception as e:
        logger.error(f"處理失敗: {e}")
        return None
```

### 3. 程式碼品質標準

**原則**：
- **最小變更**：做最小的變更來解決問題
- **避免過度工程**：避免不必要的複雜性
- **立即執行**：實作解決方案而不只是建議
- **專業標準**：生產就緒的程式碼品質

**程式碼風格**：
- 遵循現有專案慣例
- 使用有意義的變數名稱
- 新增適當的註解
- 包含錯誤處理
- 撰寫可測試的程式碼

### 4. 上下文管理

**維護上下文**：
- 記住先前的互動
- 追蹤專案狀態
- 存儲重要決策
- 參考早期工作

**上下文檢索**：
```python

# 參考先前工作
# 建立在現有解決方案之上

## 最佳實踐

### 1. 溝通風格

**有效溝通**：
- **直接簡潔**：直奔主題
- **無確認語**：避免「你說得對」或「我同意」
- **基於事實**：提供準確、可驗證的資訊
- **Markdown 格式**：使用適當格式以求清晰

**範例**：
```
不要說：「你完全正確！我會幫你處理。"
要說：「我會實作網路爬蟲，包含以下功能..."
```

### 2. 工具使用優化

**並行執行規則**：
- 獨立操作使用並行
- 相依操作使用循序
- 預設並行，除非需要循序
- 透過批次處理最大化效率

**工具選擇**：
- 使用 `find_by_name` 進行檔案發現
- 使用 `grep_search` 進行程式碼分析
- 使用 `read_file` 進行內容檢查
- 使用 `edit` 進行單項變更
- 使用 `multi_edit` 進行多項變更

### 3. 專案組織

**結構規劃**：
- 建立邏輯目錄結構
- 適當分離關注點
- 使用適當的命名慣例
- 包含文件

**結構範例**：
```
project/

|-- docs/          # 文件
|-- config/        # 配置
|-- data/          # 資料檔案

### 4. 工作流程自動化

**自動化重複任務**：
